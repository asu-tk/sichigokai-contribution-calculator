# 七護會 任意献金目安額 計算ツール

消防団の既存Excel出欠表をブラウザ内で読み込み、点検・訓練・火災出動の回数から、団員ごとの活動貢献率と「七護會 任意献金目安額」を算出する静的Webアプリです。

公開URL:

https://asu-tk.github.io/sichigokai-contribution-calculator/

アップロードされたExcelや計算結果はサーバーへ送信・保存しません。DB、Supabase、ログイン機能は使いません。

## セットアップ

```bash
cd tools/nanagokai-contribution-calculator
npm install
npm run dev
```

ブラウザで表示されたローカルURLを開きます。

## ビルド

```bash
npm run build
```

ビルド結果は `dist/` に出力されます。

## 公開方法

### GitHub Pages

1. `npm run build` を実行します。
2. 生成された `dist/` の中身を GitHub Pages の公開対象にします。
3. このアプリは `vite.config.ts` で `base: './'` にしているため、サブディレクトリ公開にも対応しやすい構成です。

### Vercel

1. Vercelでこのフォルダをプロジェクトルートとして指定します。
2. Framework Preset は `Vite` を選びます。
3. Build Command は `npm run build`、Output Directory は `dist` を指定します。

## 対応Excel形式

- 月別シート構成に対応します。
- シート名は `4月`、`５月`、`11月`、`１１月` のような表記を想定しています。
- 全角数字は半角数字に変換して月判定します。
- 前期は4月〜9月、後期は10月〜3月、年間合計は両方を集計します。

各月シートの想定構造:

| 行 | 内容 |
| --- | --- |
| 1行目 | タイトル |
| 2行目 | 活動名 |
| 3行目 | 種別。「点検」「訓練」「火災」の列のみ対象 |
| 4行目 | 月日 |
| 5行目 | 人員合計 |
| 6行目以降 | 団員別出欠 |

- A列はNO、B列は団員名、C列以降が活動ごとの出欠欄です。
- 「計」列、空欄列、3行目に対象種別がない列は無視します。
- B列が空欄の行は無視します。
- 出欠欄は数値 `1` または文字列 `"1"` のときだけ出席・出動扱いにします。

## 計算ルール

初期設定:

- 点検: 1回1ポイント
- 訓練: 1回2ポイント
- 火災出動: 1回3ポイント
- 火災ポイント上限: 半期ごとに10ポイント
- 最大目安額: 45,000円

計算式:

- 半期基本活動ポイント = 対象期間内の点検回数 × 点検P + 訓練回数 × 訓練P
- 個人獲得ポイント = 点検出席回数 × 点検P + 訓練出席回数 × 訓練P + 火災出動回数 × 火災P
- 火災ポイントは半期ごとに上限を適用します。
- 年間合計では前期・後期それぞれに火災上限を適用してから合算します。
- 活動貢献率 = 個人獲得ポイント ÷ 基本活動ポイント × 100
- 活動貢献率が100%を超える場合は100%表示にします。
- 基本活動ポイントが0の場合、活動貢献率は0%です。

七護會 任意献金目安額:

| 活動貢献率 | 七護會 任意献金目安額 |
| --- | ---: |
| 80%以上 | 0円 |
| 60%以上80%未満 | 10,000円 |
| 40%以上60%未満 | 20,000円 |
| 20%以上40%未満 | 35,000円 |
| 20%未満 | 45,000円 |

## 出力

- CSVダウンロード
- Word（docx）ダウンロード
- 印刷用表示

Wordには、タイトル、作成日、対象期間、計算ルール、サマリー、団員別集計表、注意書きを出力します。

注意書き:

> 本資料に記載された金額は、七護會への任意献金を検討するための目安額であり、確定した請求額ではありません。

## 注意事項

- このツールは任意献金を検討するための計算機です。
- 団員名簿や個人情報を保存・管理しません。
- Excelファイルはブラウザ内で読み取り、計算結果もブラウザ内だけで表示します。
- 公開時は静的ファイルとして配信できます。
